Parameters
NailsiNails around the circle. More nails = finer detail but more winding time. Fewer lines are needed per nail when nail count is high. Typical: 150–300 for 30cm board.
320
Max linesiTotal thread segments. More = denser darker image. The algorithm stops early if fully covered. Rule of thumb: ~12–15× the nail count is usually enough for a detailed portrait.
5600
Line darknessiHow much each drawn line "uses up" the darkness beneath it. Higher = even coverage. Lower = builds up very dark in shadow areas. Works best when related to Max lines — more lines needs less darkness per line.
35
Thread thicknessi0.1–0.3 = fine thread · 0.4–0.8 = embroidery floss · 1.0+ = thick yarn. Lines layer up, so thinner looks most realistic.
0.4
Min nail skipiSkip at least this many nails between connections. Prevents short ugly lines at edges. ~8% of nail count is a good rule (e.g. 20 for 250 nails). Higher = all lines are longer chords.
20
Thread Color
Nail markersiSmall dots at each nail position. Doesn't affect export.
AlgorithmiGreedy: at each step, samples every possible next nail by tracing its pixels — always picks the darkest chord. Simple and effective.

Radon: same greedy loop, but uses a different error image — a backprojection of edge-enhanced image data. Tends to produce crisper lines and better contrast in portraits.
